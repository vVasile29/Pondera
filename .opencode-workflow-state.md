### 4. Route definitions (POST / PUT / DELETE for metrics)

Add three new routes to `routers/api.py`, placed after the existing `GET /api/metrics` handler.

**POST `/api/metrics`** — Create a new metric:
- Request body: `{ "name": string, "category": string, "description": string }`

  - `name` — required, non-empty string after strip.
  - `category` — required, non-empty string after strip (any value accepted, no enum restriction).

**PUT `/api/metrics/{metric_id}`** — Update an existing metric:
- Path param: `metric_id` (int)
- Request body: `{ "name": string, "category": string, "description": string }`
- Validation: same rules as create (name required non-empty, category required non-empty).
- Duplicate name (if changing name to one that already exists on a DIFFERENT metric) → 422.
- On success: return `{ "id": int, "name": str, "category": str, "description": str }`.
- If `metric_id` not found → 404.

**DELETE `/api/metrics/{metric_id}`** — Delete a metric:
- Path param: `metric_id` (int)
- Cascade deletes `AlternativeScore` and `DecisionWeight` rows via FK `ondelete="CASCADE"`.
- On success: return `{"status": "deleted"}`.
- If `metric_id` not found → 404.
- No removal of `Decision.ko_criteria` JSON entries referencing this metric (defensive: `sanitize_persisted_ko_criteria` already skips stale metric IDs). Same for `Decision.thresholds`.
