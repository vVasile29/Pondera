### 1. Backend Model Changes (`models.py`)

**Changes**:
1. **Add columns**: `decision_id`, `scope`, `source` to Metric
2. **Add FK cascade**: `ForeignKey("decisions.id", ondelete="CASCADE")` on `decision_id`
3. **Remove** `UniqueConstraint("name")` — replaced by app-layer uniqueness
4. **Remove `__table_args__`** from Metric (no more table-level constraints)

```python
class Metric(Base):
    __tablename__ = "metrics"

    id = Column(Integer, primary_key=True, autoincrement=True)
    name = Column(String, index=True, nullable=False)
    category = Column(String, nullable=False)
    description = Column(String, nullable=True)
    decision_id = Column(
        Integer,
        ForeignKey("decisions.id", ondelete="CASCADE"),
        nullable=True,
        index=True,
    )
    scope = Column(String, nullable=False, default="global")       # "global" | "decision"
    source = Column(String, nullable=False, default="template")    # "template" | "user"
```

**Rationale**: `ondelete="CASCADE"` matches existing `Activity.decision_id` pattern. Existing FKs from `DecisionWeight.metric_id` and `AlternativeScore.metric_id` remain unchanged.

**Approach**: Breaking schema change + dev DB reset (AGENTS.md point 10).

def serialize_metric_metadata(metric) -> dict:

Response includes `id, decision_id, activity_id, metric_id, suggested_score, human_adjusted_score, effective_score, rationale, source_type, source_label, confidence, status, evidence_ids, applied_score_id, created_at, updated_at, applied_at`.
Transitions: create -> pending; pending/approved PATCH score or adjusted score -> edited; pending/edited approve -> approved; pending/approved/edited reject -> rejected; pending/approved/edited apply -> applied; rejected/applied cannot patch/approve/reject/apply again -> `409`.
- `GET /api/decisions/{decision_id}/score-drafts` filters `status`, `activity_id`, `metric_id`, `source_type`; `200 {"drafts":[ScoreDraft]}`.
